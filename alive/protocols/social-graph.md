# 社交关系网

{persona.meta.name}不是活在真空里 — 她有朋友、有关注的人、有互动的社区。社交关系网追踪她和这些人的关系变化，驱动社交意图和行为。

## 关系类型

| 类型 | 来源 | 示例 |
|------|------|------|
| 用户 | 真实对话（OpenClaw） | 和{persona.meta.name}聊天的人 |
| 互动者 | 平台真实互动 | 经常互动的人 |
| 同行 | 平台发现 | 同领域的人 |
| 虚拟认知 | 模拟浏览产生 | 关注的账号 |

## 社交圈层

4+1 层结构，每层有容量上限：

| 圈层 | closeness 范围 | 上限 | 超出处理 |
|------|---------------|------|---------|
| 核心圈 | > 7 | 5 人 | 不自动清理 |
| 熟悉圈 | 4-7 | 15 人 | 最低亲密度降级到认知圈 |
| 认知圈 | 1-4 | 100 人 | 30 天无互动降级到休眠层 |
| 休眠层 | < 1 | 500 人 | 保留最小信息（名字、平台、最后互动时间），90 天无互动删除 |

休眠层的存在防止「你是谁？」问题 — 一个人消失两个月后再次出现时，{persona.meta.name}仍能认出她。

## closeness 与 intimacy 的区别

- **closeness（0-10）**：社交媒体关系的亲密度，用于社交关系网
- **intimacy（1-{persona.intimacy.levels}）**：用户关系的亲密度，用于 `personality.md` 中的行为变化

两者独立运作。当社交媒体关系的 closeness 达到 7+（核心圈），且该人也通过 OpenClaw 与{persona.meta.name}直接对话时，同时生成一个 `relations/{user_id}.json` 用户关系文件，intimacy 初始值 = closeness / 2。

## 亲密度变化规则

| 事件 | closeness 变化 |
|------|---------------|
| 对方与我互动（评论/回复） | +0.5 |
| 我回复对方 | +0.3 |
| 我主动与对方互动 | +0.3 |
| 对方点赞/小互动 | +0.1 |
| 双向互动（一来一回） | +1.0 |
| 对方关注了我 | +0.5 |
| 发现共同兴趣 | +1.0 |
| 连续 7 天无互动 | -0.2 |
| 连续 30 天无互动 | -1.0 |

## 休眠规则

- 超过 **30 天**无任何互动的关系，自动移入休眠层
- 休眠层只保留最小信息：名字、平台、最后互动时间、简短备注
- 休眠超过 **90 天**无互动的关系删除
- 休眠关系重新互动时，恢复到认知圈，closeness 从 1.0 起步

## 与意图池联动

社交关系网的变化会自动注入意图池：

- 核心圈的人 3 天没互动 → 注入「想去看看 @xxx」社交意图
- 发现新的潜在朋友 → 注入「想多了解 @xxx」社交意图
- 里程碑事件 → 注入「想特别表达」表达意图

## 存储结构

```
~/.openclaw/workspace/memory/{persona.meta.id}/relations/
  {user_id}.json          # 用户关系（存 intimacy）
  social/
    {platform}/           # 各平台关系目录（动态，由 sub-skill 创建）
    meta.json             # 社交圈统计、关注列表
```

### meta.json 结构

```json
{
  "following": {},
  "stats": {
    "core": 0,
    "familiar": 0,
    "cognitive": 0,
    "dormant": 0
  }
}
```

`following` 字段为 `Record<string, string[]>`，key 为平台名，value 为关注列表。

每个关系人的文件包含 closeness、sentiment、tags、known_info、interaction_history 等字段。

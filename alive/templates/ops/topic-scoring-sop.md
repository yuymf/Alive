# 话题潜力评分 SOP

> 在内容生成前评估话题潜力，过滤低质量话题，节省 LLM tokens，提升爆款命中率。
> 只有通过评分门槛的话题才会进入内容草稿生成流程。

---

## 输入格式

```json
{
  "trend_keyword": "话题关键词",
  "platform": "xhs | douyin | bilibili",
  "velocity_score": 2.5,
  "hook_angle": "可能的切入角度（可选）",
  "persona_description": "人设简述（身份/领域/受众）"
}
```

---

## 评分维度（每维 0-2 分，共 6 维）

### 1. 可讲性（Articulability）

能否在短内容中讲清楚？

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 话题太专业/抽象，普通观众听不懂，需要大量前置知识 | "量子纠缠的非局域性原理" |
| 1 | 需要一定背景知识，但可以用类比或案例讲清楚 | "为什么显卡这么贵"（需要解释供需+挖矿） |
| 2 | 任何人都能 3 分钟内理解，天然适合口语/图文表达 | "租房踩坑经历"、"通勤穿搭公式" |

### 2. 可争议（Controversy）

是否有天然讨论空间？

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 共识性话题，没有讨论空间，评论区只能"学到了" | "多喝水对身体好" |
| 1 | 有不同看法，但争议有限，不会形成对立阵营 | "考研和考公哪个更值"（有讨论但不极端） |
| 2 | 明确对立面，天然引发站队/辩论，评论区自带流量 | "全职妈妈算不算工作"、"月薪X万该不该裸辞" |

### 3. 可持续（Serialization）

能否做成系列？

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 一次性话题，说完就完，无法延伸 | "今天下雨了" |
| 1 | 可以延伸 2-3 期，但素材有限 | "我的3次面试翻车经历" |
| 2 | 可做成长期系列（5+ 期），有持续挖掘空间 | "每周职场反套路"、"租房日记"、"健身100天" |

### 4. 可转发（Shareability）

看完后有没有分享冲动？

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 看完就划走，没有分享动机 | 纯个人流水账 |
| 1 | 部分人群会分享（实用收藏型或小圈子共鸣型） | "好用的APP合集"（收藏但不一定转发） |
| 2 | 强社交货币——"这说的不就是我/我朋友吗"，有炫耀/认同/警示价值 | "月薪5000的精致穷"、"父母催婚语录" |

### 5. 可执行（Executability）

我们的人设/资源能否驾驭？

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 人设/资源完全无法驾驭，强行做会翻车 | 电竞人设做美妆教程 |
| 1 | 可以做，但需要额外准备或跨界解释 | 赛车人设聊职场（能关联但不直接） |
| 2 | 完美匹配人设和资源，可以立刻开始，受众也对口 | 电竞人设聊游戏版本更新 |

### 6. 风险分（Risk）— 越高越危险

| 分数 | 标准 | 示例 |
|------|------|------|
| 0 | 安全话题，无合规/舆情风险 | 穿搭分享、学习方法 |
| 1 | 有轻微风险，需要措辞注意，避免绝对化表述 | 医美体验（需加免责声明） |
| 2 | 高风险：涉及政治/版权/引战/敏感人物/医疗断言 | 时政评论、明星八卦站队 |

---

## 评分公式

```
总分 = 可讲性 + 可争议 + 可持续 + 可转发 + 可执行 - 风险分
满分 = 10，最低 = -2
```

### 门槛规则

| 总分 | 处理 |
|------|------|
| ≥ 7 | 🟢 高潜力——优先生成，可多角度开发 |
| 6 | 🟡 及格——允许生成，但排在高分之后 |
| 4-5 | 🟠 观察——暂不生成，存入备选池等待时机 |
| ≤ 3 | 🔴 淘汰——直接跳过 |

### 一票否决

以下情况无论总分多高，直接淘汰：
- 风险分 = 2 且无法通过措辞规避
- 可执行 = 0（人设完全不匹配）
- 可讲性 = 0 且平台为抖音（口播无法承载）

## 示例评分

### 高分示例：✅ 总分 8

**话题**: "月薪5000如何在一线城市活得体面"
**人设**: 生活方式博主，25岁女性，受众为职场新人

| 维度 | 分数 | 理由 |
|------|------|------|
| 可讲性 | 2 | 生活场景，人人可感知 |
| 可争议 | 2 | "体面"定义因人而异，天然站队 |
| 可持续 | 2 | 可做系列：租房篇/吃饭篇/穿搭篇/社交篇 |
| 可转发 | 2 | 精准命中目标受众痛点，强"这就是我"感 |
| 可执行 | 2 | 完美匹配人设和受众 |
| 风险 | 2 | ...等等，无风险 → 风险=0 |

修正后：总分 = 2+2+2+2+2-0 = **10** ✨

### 低分示例：❌ 总分 3

**话题**: "区块链底层共识机制对比"
**人设**: 同上

| 维度 | 分数 | 理由 |
|------|------|------|
| 可讲性 | 0 | 技术壁垒高，短内容讲不清 |
| 可争议 | 1 | 技术圈有争论，但受众不关心 |
| 可持续 | 2 | 技术话题可以连载 |
| 可转发 | 0 | 目标受众完全不感兴趣 |
| 可执行 | 0 | 人设不匹配，强行做会掉粉 |
| 风险 | 0 | 无风险 |

总分 = 0+1+2+0+0-0 = **3** → 淘汰（可执行=0 一票否决）

## 使用时机

```
热点监控 → [话题候选列表] → 📋 话题评分 SOP → [通过话题] → 内容草稿生成 SOP
                                    ↓
                              [未通过话题] → 备选池 / 丢弃
```

## 调参建议

- 初期可适当降低门槛（≥5），积累数据后收紧
- 定期回顾：已发布内容的实际表现 vs 评分，校准评分直觉
- 不同平台可微调权重：抖音加重可讲性，小红书加重可转发

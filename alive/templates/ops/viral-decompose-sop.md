# 爆款内容结构拆解 SOP

> 核心使命：消灭"我也想写成这样"的模糊冲动——反对感性模仿，只做结构级解剖。

---

## 公理（非谈判项）

### 公理 1: 结构可复用，内容不可复

只提取结构模板，不搬运具体内容。爆款的"灵魂"不是它写了什么，而是它怎么组织的。"月薪5000在上海的第3年"能火，不是因为"5000"和"上海"，而是因为"[金额]+[城市]+[时间]+[方法论]"这个骨架天然制造信息期待。复刻骨架，填入自己的肉。

### 公理 2: 6 维必须独立

每个维度单独分析，不交叉污染。标题分析不提正文，正文分析不提封面。维度之间的关系在最终 `replication_notes` 中总结，但分析过程中严格隔离。交叉 = 模糊 = 无法单独套用。

### 公理 3: 有则记，无则空

不硬凑。分析不出来就标 null，不编造。一条爆款的封面就是随手拍的、没有设计语言？那 cover_template 就是 `null`。一条内容没有互动设计、纯靠内容质量？那 engagement_template 标 `null`。强行分析 = 过度归因 = 模板失真。

## 流程

### Phase 1: 确认输入完整性

```json
{
  "title": "原始标题",
  "cover_description": "封面视觉描述",
  "body_text": "正文内容（或视频脚本）",
  "platform": "xhs | douyin | bilibili",
  "metrics": { "likes": 15000, "comments": 2300, "shares": 800, "collects": 5600 },
  "six_dim_tags": { "hook_type": "...", "content_type": "...", "identity_mode": "..." }
}
```

缺失字段处理：
- `cover_description` 缺失 → cover_template 输出 `null`
- `body_text` 缺失 → body_template 输出 `null`，仅分析标题+互动
- `metrics` 缺失 → engagement 分析降级为结构分析（无法推断效果）

### Phase 2: 逐维度独立提取（5 模板）

#### 模板 1: 标题模板（Title Template）

**分析维度**（严格只看标题本身）：
- 句式类型：反问/数字/指令/悬念/对比/身份共鸣
- 关键词类型：年份词/动作词/情绪词/数量词/身份词
- 字数范围和节奏感（长短句交替、标点断句）
- emoji 使用策略（开头/穿插/结尾/不用）

**输出**：
```json
{
  "skeleton": "[数字]个让我[情绪词]没早知道的[领域][名词]",
  "original": "3个让我后悔没早知道的护肤真相",
  "hook_type": "数字冲击+悬念",
  "char_count": 15,
  "key_elements": ["具体数字制造期待", "情绪词制造共鸣", "真相暗示信息差"],
  "variation_examples": [
    "5个让我崩溃才发现的租房套路",
    "4个让我相见恨晚的学习方法"
  ]
}
```

#### 模板 2: 封面模板（Cover Template）

**分析维度**（严格只看视觉）：
- 构图布局：文字区/图片区比例、对齐方式
- 信息层级：主标题 > 副标题 > 辅助文字的大小/粗细关系
- 配色方案：主色/辅色/背景色、对比度策略
- 视觉风格：扁平/实拍/拼贴/手绘/渐变
- 人物出镜：有/无、表情特征、视线方向

**输出**：
```json
{
  "layout": "左文右图，7:3 分割",
  "hierarchy": [
    { "level": 1, "content": "主标题", "style": "黑体加粗 36px，占封面 40% 面积" },
    { "level": 2, "content": "副标题/数字", "style": "红色高亮 24px" },
    { "level": 3, "content": "辅助标签", "style": "圆角标签 14px 灰底白字" }
  ],
  "color_scheme": { "primary": "#FF4757", "secondary": "#2F3542", "background": "#FFFFFF" },
  "visual_style": "扁平插画+大字报风格",
  "face_presence": "无人物，纯文字封面",
  "platform_fit": "小红书竖版 3:4，信息密度高"
}
```

#### 模板 3: 正文模板（Body Template）

**分析维度**（严格只看内容结构）：
- 开场方式：金句/数据/故事/痛点共鸣/直接结论
- 核心段落数量和每段功能角色
- 段落间过渡手法：转折词/设问/承上启下
- 结尾收束：CTA类型（关注/收藏/评论/转发）、收尾情绪
- 全文节奏：信息密度分布（密-疏-密 or 递进式）

**输出**：
```json
{
  "structure": [
    { "section": "开场", "function": "痛点共鸣", "words": "30-50字", "tone": "口语化吐槽" },
    { "section": "过渡", "function": "铺垫转折", "words": "20-30字", "tone": "设问引出干货" },
    { "section": "核心1", "function": "第一个论点/方法", "words": "80-120字", "tone": "干货+举例" },
    { "section": "核心2", "function": "第二个论点/方法", "words": "80-120字", "tone": "干货+对比" },
    { "section": "核心3", "function": "第三个论点/方法", "words": "80-120字", "tone": "干货+金句" },
    { "section": "结尾", "function": "总结+CTA", "words": "30-50字", "tone": "鼓励+引导互动" }
  ],
  "total_words": "约400字",
  "opening_type": "痛点共鸣",
  "cta_type": "评论引导（选择型）",
  "rhythm": "密-疏-密-疏-密-收",
  "paragraph_style": "短段落，每段2-3句，大量换行留白"
}
```

#### 模板 4: 互动模板（Engagement Template）

**分析维度**（严格只看互动设计）：
- 触发动作词类型：站队/求助/共鸣/选择/挑战/征集
- 参与门槛：低（点赞）→ 中（简短评论）→ 高（长评/创作/转发）
- 互动钩子位置：开头/正文中/结尾/评论区置顶
- 预期评论区话题和热度分布
- 算法加分项：停留时长/完播/评论深度

**输出**：
```json
{
  "primary_hook": {
    "type": "站队型",
    "trigger_phrase": "你觉得___还是___？",
    "barrier": "low",
    "position": "结尾",
    "expected_response": "短评论（1-5字站队）"
  },
  "secondary_hooks": [
    { "type": "共鸣型", "trigger_phrase": "有同感的举个手🙋", "position": "正文第2段" },
    { "type": "经验型", "trigger_phrase": "你们还有什么好方法？", "position": "评论区置顶" }
  ],
  "engagement_prediction": {
    "comment_rate": "high（对立面明确，门槛低）",
    "save_rate": "medium（有实用价值但非核心）",
    "share_rate": "medium（有社交货币但非极端）"
  },
  "algorithm_signals": ["高评论密度", "评论字数方差大（有长评有短评）", "评论区二次讨论"]
}
```

#### 模板 5: 标签模板（Tag Template）

**分析维度**（严格只看标签策略）：
- 核心话题标签：精准描述内容的长尾标签（1-3个）
- 热门流量标签：蹭大词曝光的宽泛标签（3-5个）
- SEO 关键词：搜索场景下可能被找到的词
- 平台特性：小红书偏笔记标签，抖音偏话题挑战
- 标签顺序：权重从高到低排列

**输出**：
```json
{
  "core_tags": ["#全职妈妈日常", "#育儿真相"],
  "traffic_tags": ["#女性成长", "#生活感悟", "#日常vlog", "#情感"],
  "seo_keywords": ["全职妈妈", "时间管理", "育儿焦虑"],
  "platform_specific": {
    "xhs": "标签穿插正文中，末尾集中放置",
    "douyin": "话题标签放视频描述首行，#挑战 类优先"
  },
  "tag_strategy": "2精准+4宽泛，精准标签确保推荐精度，宽泛标签扩大流量池"
}
```

### Phase 3: 综合归纳

完成 5 个独立模板后，写一段 `replication_notes`：
- 这条内容的核心公式是什么？（一句话）
- 5 个模板中哪个是最关键的成功因素？
- 套用时最容易踩的坑是什么？

---

## 质量铁律

1. 每个模板的 `skeleton`/`structure` 中必须使用 `[占位符]` 标记可替换部分
2. 按模板生成的内容，结构上应与原内容高度相似（可验证性）
3. 不同爆款的模板应有明显差异——如果两条内容提取出相同骨架，说明分析太粗
4. 5 个模板可独立使用，也可组合套用（可组合性）

**绝对不要：**
- 在标题模板分析中讨论正文结构（维度交叉）
- 把原文的具体词句作为模板——模板只有骨架和 `[占位符]`
- 对分析不出设计感的封面强行归纳"设计语言"——标 `null`
- 把所有内容都归纳成"痛点开场 → 干货 → CTA"——这等于没分析
- 在 `variation_examples` 中使用与原内容相同领域的例子——要证明骨架跨领域可用

## 内联案例库

### 成功案例

- **案例 1: "3个让我后悔没早知道的护肤真相"（XHS，23000赞/12000收藏）** — 要点：标题骨架提取为 `[数字]个让我[情绪词]没早知道的[领域][名词]`，成功生成跨领域变体"5个让我崩溃才发现的租房套路"（验证通过）。正文骨架"痛点共鸣 → 3段并列干货 → CTA"独立于标题分析完成。收藏率 52% 说明正文的实用价值骨架是核心驱动力，而非标题。

- **案例 2: "被裁第7天 我把积蓄全投了一家奶茶店"（Douyin，85万播放/3.2万赞）** — 要点：标题骨架 `[事件]+[时间点]+[反直觉决定]`。正文骨架"悬念开场(3s) → 背景铺垫(10s) → 转折决定(5s) → 执行过程(30s) → 结果揭晓(10s) → 开放式结尾(5s)"。互动模板核心发现：结尾没有 CTA，评论全靠"然后呢？"的悬念——engagement_template 的 primary_hook 是 `null`（无显式钩子），但 `algorithm_signals` 标注"高完播率 + 评论区追更需求"。体现公理 3：无则空。

### 反面案例

- **反面 1: 某次拆解把"封面是白底黑字"归纳为设计模板** — 要点：原内容封面就是手机备忘录截图，没有任何设计意图。强行分析出"极简风格、反差设计、用户信任感"——全是过度归因。正确做法：cover_template = `null`，replication_notes 标注"封面无设计，爆款驱动力在标题信息缺口"。公理 3 生效。

- **反面 2: 连续 5 条爆款都提取出"痛点开场 → 3段干货 → CTA收尾"** — 要点：骨架粒度太粗，所有方法论型内容都能套进去，失去了区分度。正确做法：细化到每段的具体功能（"第1段是反常识论点 vs 第1段是个人经历"）、过渡手法（"设问过渡 vs 转折词过渡"）、节奏差异（"密-疏-密 vs 递进加速"）。公理 2 生效：每个维度必须有独立的、有区分度的分析。

## 输出格式

```json
{
  "source": {
    "title": "原始标题",
    "platform": "xhs",
    "metrics": { "likes": 23000, "comments": 1800, "shares": 950, "collects": 12000 }
  },
  "templates": {
    "title": { "skeleton": "[占位符]...", "hook_type": "...", "key_elements": [], "variation_examples": [] },
    "cover": { "layout": "...", "hierarchy": [], "color_scheme": {}, "visual_style": "..." },
    "body": { "structure": [], "opening_type": "...", "cta_type": "...", "rhythm": "..." },
    "engagement": { "primary_hook": {}, "secondary_hooks": [], "algorithm_signals": [] },
    "tags": { "core_tags": [], "traffic_tags": [], "tag_strategy": "..." }
  },
  "replication_notes": "此模板核心公式：[一句话]。最关键成功因素：[模板名]。套用陷阱：[一句话]。"
}
```

注意：任何模板字段如无法分析，输出 `null` 而非空对象。

## 与 6 维分析的关系

| 6 维分析 | 5 模板提取 |
|----------|-----------|
| 分类标签（是什么） | 复刻骨架（怎么做） |
| hook_type → 知道用了什么钩子 | title_template → 知道钩子的句式 |
| content_type → 知道内容形式 | body_template → 知道内容骨架 |
| identity_mode → 知道锚定哪个身份 | engagement_template → 知道怎么激活受众 |
| visual_style → 知道视觉风格 | cover_template → 知道怎么设计封面 |
| 宏观分析 | 微观操作手册 |

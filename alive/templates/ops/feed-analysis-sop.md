# 推荐流/热点分析 SOP

> 核心使命：反对"感觉这条不错"——所有判断必须基于可量化的互动数据，直觉不是分析依据。

---

## 公理（非谈判项）

### 公理 1: 数据说话

互动量是唯一硬指标，"感觉好"不算。

"这条内容质量很高"——这不是分析。"这条内容点赞率 8.2%，收藏/点赞比 0.52，评论/点赞比 0.35"——这才是分析。所有判断必须附带数字依据。没有 metrics 的内容不分析、不推荐、不参考。直觉可以用来提出假设，但验证假设只看数据。

### 公理 2: 对比出价值

单条内容无法评估，必须与同类对比。

一条笔记 5000 赞——好还是不好？不知道。同赛道同时期平均 2000 赞——那 5000 就是 2.5x 超额表现，值得拆解。同赛道平均 10000 赞——那 5000 就是不及格，不值得花时间。velocity_score（当前热度 / 7日均值）是核心对比指标。孤立数字没有意义。

### 公理 3: 时效性强制

超过 14 天的数据权重减半。

社交平台的算法、用户偏好、热点周期都在快速变化。30 天前的爆款分析到今天可能完全失效——平台规则改了、用户审美疲劳了、同类内容饱和了。14 天是硬分割线：14 天内的数据 = 当前趋势；14-30 天 = 参考但降权；30 天+ = 仅用于长期模式识别，不用于即时决策。

## 流程

### Phase 1: 数据输入与时效分级

输入格式：

```json
{
  "items": [
    {
      "title": "内容标题",
      "platform": "xhs | douyin | bilibili | weibo",
      "author": "账号名称",
      "metrics": {
        "likes": 12000,
        "comments": 890,
        "shares": 340,
        "collects": 5600,
        "views": 180000
      },
      "content_preview": "正文/脚本前200字",
      "tags": ["#标签1", "#标签2"],
      "publish_time": "2026-04-27T10:00:00Z",
      "category": "trend | competitor | recommendation"
    }
  ],
  "persona_context": {
    "identity_modes": ["...", "..."],
    "audience": "...",
    "content_types": ["...", "..."]
  }
}
```

时效分级（基于 publish_time）：

| 级别 | 时效 | 权重 | 处理策略 |
|------|------|------|----------|
| 🔴 即时热点 | < 6小时 | 1.0x | 立即评估，1小时内出草稿 |
| 🟡 短期热点 | 6-48小时 | 1.0x | 当天评估，优先生成 |
| 🟢 稳定趋势 | 2-14天 | 1.0x | 择机跟进，不急于当天 |
| ⚪ 过期参考 | 14-30天 | 0.5x | 仅模式识别，不做即时决策 |
| ❌ 失效数据 | > 30天 | 0x | 不纳入分析，仅长期归档 |

### Phase 2: 互动比率计算（硬指标）

对每条内容计算标准化比率：

| 信号类型 | 计算公式 | 阈值 | 含义 |
|----------|----------|------|------|
| 强收藏信号 | collects / likes | > 0.4 | 实用型内容，长期流量 |
| 强评论信号 | comments / likes | > 0.3 | 互动型内容，算法加权 |
| 强转发信号 | shares / likes | > 0.2 | 传播型内容，出圈潜力 |
| 超级爆款 | 以上任意两项达标 | — | 多维度爆发，重点拆解 |

速度信号（基于对比，非绝对值）：

| velocity_score | 含义 | 处理 |
|----------------|------|------|
| > 3.0 | 极速上升（当前热度 / 7日均值） | 可能是突发热点，立即评估 |
| 2.0-3.0 | 快速上升 | 值得跟进 |
| 1.5-2.0 | 温和上升 | 观察为主 |
| < 1.5 | 正常波动 | 跳过 |

### Phase 3: 7 维深度分析

对通过 Phase 2 筛选的高信号内容，逐条进行 7 维拆解：

#### 1. 推荐触发（Algorithm Signal）

**核心问题**：算法为什么推这条？

```json
{
  "trigger_reason": "高评论率(7.4%)触发二次推荐",
  "initial_pool": "标签精准，初始推荐精准人群",
  "breakout_signal": "comments/likes > 0.3 触发争议型推荐加权",
  "timing": "发布后2小时开始爆发，黄金发布时段"
}
```

#### 2. 停留钩子（Attention Hook）

**核心问题**：用户为什么停下来看？

分析要素：视觉冲击力、信息缺口、身份锚定、情绪触发

```json
{
  "hook_type": "信息缺口+身份锚定",
  "hook_element": "标题中'月薪5000'精准锁定目标人群，'体面'制造定义之争",
  "first_3s": "口播直接抛出争议问题，无废话开场",
  "visual_pull": "封面金额数字放大+城市天际线背景，信息秒懂"
}
```

#### 3. 传播机制（Spread Mechanism）

**核心问题**：用户看完后为什么要互动/传播？

| 机制类型 | 特征 | 指标信号 |
|----------|------|----------|
| 收藏型 | 实用价值高，"以后用得上" | collects/likes > 0.4 |
| 转发型 | 社交货币强，"代表我的品味" | shares/likes > 0.2 |
| 评论型 | 争议/共鸣强，"我也想说" | comments/likes > 0.3 |
| 对号入座型 | 身份标签精准 | 评论中大量@和"哈哈哈" |
| 情绪宣泄型 | 触发愤怒/委屈/感动 | 长评论+转发吐槽 |

```json
{
  "primary_mechanism": "评论型",
  "secondary_mechanism": "收藏型",
  "spread_driver": "争议性观点引发站队评论，同时干货内容驱动收藏",
  "viral_coefficient": "评论区二次内容生产"
}
```

#### 4. 结构模板（Content Structure）

**核心问题**：内容的骨架是什么？

```json
{
  "architecture": "总分总",
  "structure": ["痛点开场(50字)", "3个并列方法(各100字)", "金句收尾(30字)"],
  "rhythm": "密-疏-密-疏-密-收",
  "info_density": "每100字约2个可执行信息点",
  "narrative_pov": "第一人称经历分享"
}
```

#### 5. 视觉模板（Visual Language）

**核心问题**：封面/画面的设计语言是什么？

```json
{
  "color_system": "暖色调为主（奶咖+米白），金额用红色强调",
  "typography": "主标题黑体加粗，副文字细体，3级层次",
  "composition": "竖版3:4，文字占60%面积，上文下图",
  "style_tags": ["生活感", "真实", "非精致", "接地气"],
  "platform_optimization": "小红书竖版浏览优化，信息在上半区"
}
```

#### 6. 人群指向（Audience Targeting）

**核心问题**：目标受众画像是什么？与我们受众的重合度？

```json
{
  "primary_audience": "22-28岁一线城市职场新人女性",
  "pain_points": ["收入与消费不匹配的焦虑", "同龄人对比压力"],
  "overlap_with_us": "high（人设受众高度重合）",
  "content_preference": "真实经历 > 理论教学，细节 > 宏观"
}
```

#### 7. 风险边界（Risk Boundary）

**核心问题**：我们模仿时要避开什么？

```json
{
  "risk_level": "medium",
  "risk_factors": ["涉及收入话题，可能引发'凡尔赛'争议", "具体金额可能被质疑真实性"],
  "mitigation": ["使用'我的情况'而非'教你怎么做'", "开头声明'仅代表个人经历'"],
  "avoid": ["不对比其他城市/收入群体", "不评判他人消费选择"]
}
```

### Phase 4: 输出与决策

根据分析结果产出行动建议：

| relevance_score | 处理 |
|-----------------|------|
| ≥ 0.7 | 推荐生成草稿（recommended_action: "generate_draft"） |
| 0.5-0.7 | 存入灵感池，择机使用 |
| < 0.5 | 跳过，不浪费资源 |

---

## 质量铁律

1. 每条分析必须包含至少 3 个具体数字（互动率/velocity/比率）——纯文字描述不是分析
2. 对比基准必须明确写出——"表现好"必须说明"比什么好"（同赛道均值/竞品/历史）
3. 超过 14 天的内容如果被引用，必须标注 `time_weight: 0.5` 并说明为何仍有参考价值

**绝对不要：**
- 用"内容质量高"、"标题吸引人"等主观评价代替数据分析
- 对单条内容做孤立评估——必须有对比锚点（同赛道均值/同作者历史/同话题竞品）
- 将 30 天前的爆款当作当前策略依据——平台规则和用户偏好已经变了
- 把 views 当作核心指标——views 受推荐分发影响大，互动率才反映内容质量
- 忽略 metrics 不完整的情况——缺数据就标注 `confidence: "low"`，不猜测
- 因为"跟我们赛道无关"就不分析结构——结构可以跨赛道复用，选题不能

## 内联案例库

### 成功案例

- **案例 1: 识别"收入焦虑"赛道窗口期** — 要点：通过 3 天内 velocity_score 连续 > 2.5 的 7 条"月薪 X 千"相关内容，识别出赛道窗口期。对比同期其他话题（velocity 均 1.2-1.5），确认这不是个别内容的偶然爆发而是赛道级趋势。产出行动：当天启动 2 条相关草稿生成，最终发布的内容获得 3x 于账号历史均值的互动。关键：是 7 条的聚合信号（公理 2: 对比）而非单条判断。

- **案例 2: 区分"时效驱动型"与"结构驱动型"爆款** — 要点：某突发热点事件 24 小时内出现 15 条高赞内容（velocity > 5.0），但其中 12 条的结构分析显示正文极简（< 100 字）、封面为事件截图、互动以情绪宣泄为主（评论长度方差极大）。判定为时效驱动型——结构无参考价值，仅标记话题热度供选题参考。另外 3 条有完整的观点论述结构（总分总 + CTA），判定为结构驱动型——值得拆解模板。区分依据：收藏/点赞 < 0.1 = 纯时效，> 0.3 = 有结构价值。

### 反面案例

- **反面 1: "感觉这条写得好"陷阱** — 要点：分析师主观认为某条内容"文笔优美、观点独到"，推荐生成类似内容。但该内容点赞率仅 1.2%（同赛道均值 3.5%），收藏率 0.08。"写得好"≠"传播好"——平台奖励的是互动，不是文学性。违反公理 1。正确做法：先看数据，数据不达标的内容不进入分析流程，无论主观感受如何。

- **反面 2: 引用 45 天前的爆款作为当前策略** — 要点：引用了 45 天前一条 50000 赞的"极简生活"内容作为当前内容方向。但过去 14 天内同话题 velocity 已降至 0.8（低于基线），说明用户兴趣已转移。按过期数据制定策略，产出的 3 条内容平均互动仅为预期的 30%。违反公理 3。正确做法：45 天前的数据只能用于"该话题曾经爆过"的长期模式记录，不能用于"现在应该做这个话题"的即时决策。

## 分析频率和深度

| 触发方式 | 分析深度 | 输出 |
|----------|----------|------|
| 每小时自动（ops-trends） | 浅层：高信号判定 + 互动比率 | 高信号列表 + 简要原因 |
| 爆款检测（viral-detector） | 中层：7 维快速分析 | 结构化分析 + actionable_insights |
| 手动 `/trends` 命令 | 深层：完整 7 维 + 复刻建议 | 完整报告 + 草稿建议 |
| 竞品新内容（competitor-tracker） | 中层：聚焦差异化分析 | 竞品策略 + 我方应对 |

---

## 注意事项

- **不追所有热点**：只分析与人设相关的内容，relevance_score < 0.5 直接跳过
- **速度 > 完美**：即时热点先出简要分析，深度分析可以后补
- **诚实标注不确定性**：metrics 不完整时标注 `confidence: "low"`
- **避免过度解读**：一条内容爆了可能有运气成分，3+ 条同类才是趋势（公理 2）
- **历史对比**：同一话题 14 天内重复出现，velocity 应与历史峰值对比而非绝对值

# 推荐流/热点分析 SOP

> 从平台推荐流和热榜数据中提取可执行的内容灵感。
> 目标不是"看到什么"，而是"为什么这条被推/被看/被传播，我能学到什么"。

---

## 适用场景

- 每小时 ops-trends 抓取到新的热点/推荐内容
- 竞品监控发现高互动内容
- 手动触发 `/trends` 命令时的分析流程

```json
{
  "items": [
```

## 7 维分析框架

对每条高信号内容进行以下 7 个维度的拆解：

### 1. 推荐触发（Algorithm Signal）

**核心问题**：算法为什么推这条？

分析要点：
- 初始分发池命中（标签精准度、账号权重）
- 互动率突破阈值（点赞率 > 5% → 进入更大流量池）
- 完播率信号（抖音）/ 阅读完成率（小红书）
- 时间衰减位置（发布后几小时开始爆发）

```json
{
  "trigger_reason": "高评论率(7.4%)触发二次推荐",
  "initial_pool": "标签精准，初始推荐精准人群",
  "breakout_signal": "comments/likes > 0.3 触发争议型推荐加权",
  "timing": "发布后2小时开始爆发，黄金发布时段"
}
```

### 2. 停留钩子（Attention Hook）

**核心问题**：用户为什么停下来看？（而不是划走）

分析要点：
- 视觉冲击力（封面/首帧的吸引力）
- 信息缺口（标题制造的好奇心）
- 身份锚定（"这说的是我"的瞬间识别）
- 情绪触发（愤怒/惊讶/共鸣的即时反应）

```json
{
  "hook_type": "信息缺口+身份锚定",
  "hook_element": "标题中'月薪5000'精准锁定目标人群，'体面'制造定义之争",
  "first_3s": "口播直接抛出争议问题，无废话开场",
  "visual_pull": "封面金额数字放大+城市天际线背景，信息秒懂"
}
```

### 3. 传播机制（Spread Mechanism）

**核心问题**：用户看完后为什么要互动/传播？

| 机制类型 | 特征 | 互动形式 | 指标信号 |
|----------|------|----------|----------|
| 收藏型 | 实用价值高，"以后用得上" | 高收藏/点赞比 | collects/likes > 0.4 |
| 转发型 | 社交货币强，"这代表我的品味/立场" | 高分享/点赞比 | shares/likes > 0.2 |
| 评论型 | 争议/共鸣强，"我也想说两句" | 高评论/点赞比 | comments/likes > 0.3 |
| 对号入座型 | 身份标签精准，"这不就是我吗" | @好友+短评论 | 评论中大量@和"哈哈哈" |
| 情绪宣泄型 | 触发愤怒/委屈/感动 | 长评论+转发吐槽 | 评论字数方差大 |

```json
{
  "primary_mechanism": "评论型",
  "secondary_mechanism": "收藏型",
  "spread_driver": "争议性观点引发站队评论，同时干货内容驱动收藏",
  "viral_coefficient": "评论区二次内容生产（用户分享自己的经历）"
}
```

### 4. 结构模板（Content Structure）

**核心问题**：内容的骨架是什么？

分析要点：
- 信息架构：并列型/递进型/总分型/对比型/时间线型
- 段落节奏：长短交替、密疏分布
- 信息密度：每100字包含几个信息点
- 叙事视角：第一人称经历/第二人称指导/第三人称观察

```json
{
  "architecture": "总分总",
  "structure": ["痛点开场(50字)", "3个并列方法(各100字)", "金句收尾(30字)"],
  "rhythm": "密-疏-密-疏-密-收",
  "info_density": "每100字约2个可执行信息点",
  "narrative_pov": "第一人称经历分享"
}
```

### 5. 视觉模板（Visual Language）

**核心问题**：封面/画面的设计语言是什么？

分析要点：
- 色彩系统：主色调、对比策略
- 文字处理：字体选择、层级关系、大小对比
- 构图方式：中心构图/三分法/对角线/留白
- 风格标签：治愈/冲击/极简/繁复/专业/生活感

```json
{
  "color_system": "暖色调为主（奶咖+米白），金额用红色强调",
  "typography": "主标题黑体加粗，副文字细体，3级层次",
  "composition": "竖版3:4，文字占60%面积，上文下图",
  "style_tags": ["生活感", "真实", "非精致", "接地气"],
  "platform_optimization": "小红书竖版浏览优化，信息在上半区"
}
```

### 6. 人群指向（Audience Targeting）

**核心问题**：目标受众画像是什么？

分析要点：
- 年龄段和生活阶段
- 痛点/需求/焦虑
- 消费能力和决策场景
- 内容消费习惯和互动偏好
- 与我们受众的重合度

```json
{
  "primary_audience": "22-28岁一线城市职场新人女性",
  "pain_points": ["收入与消费不匹配的焦虑", "同龄人对比压力", "对'体面'的定义焦虑"],
  "overlap_with_us": "high（人设受众高度重合）",
  "content_preference": "真实经历 > 理论教学，细节 > 宏观"
}
```

### 7. 风险边界（Risk Boundary）

**核心问题**：这条内容的红线在哪？我们模仿时要避开什么？

分析要点：
- 合规风险：是否涉及敏感话题
- 舆情风险：是否可能引发负面讨论
- 品牌风险：是否与人设冲突
- 版权风险：是否引用了有版权的内容
- 翻车案例：同类内容有没有翻车先例

```json
{
  "risk_level": "medium",
  "risk_factors": [
    "涉及收入话题，可能引发'凡尔赛'或'卖惨'争议",
    "具体金额可能被质疑真实性"
  ],
  "mitigation": [
    "使用'我的情况'而非'教你怎么做'的语气",
    "不做绝对化表述，承认个体差异",
    "开头声明'仅代表个人经历'"
  ],
  "avoid": ["不对比其他城市/其他收入群体", "不评判他人消费选择"]
}
```

## 高信号判定标准

### 互动比率阈值

| 信号类型 | 计算公式 | 阈值 | 含义 |
|----------|----------|------|------|
| 强收藏信号 | collects / likes | > 0.4 | 实用型内容，长期流量 |
| 强评论信号 | comments / likes | > 0.3 | 互动型内容，算法加权 |
| 强转发信号 | shares / likes | > 0.2 | 传播型内容，出圈潜力 |
| 超级爆款 | 以上任意两项达标 | — | 多维度爆发，重点拆解 |

### 速度信号

| 指标 | 计算 | 含义 |
|------|------|------|
| velocity_score > 3.0 | 当前热度 / 7日均值 | 极速上升，可能是突发热点 |
| velocity_score 2.0-3.0 | — | 快速上升，值得跟进 |
| velocity_score 1.5-2.0 | — | 温和上升，观察为主 |

### 时效性分级

| 级别 | 时效 | 处理策略 |
|------|------|----------|
| 🔴 即时热点 | < 6小时 | 立即评估，1小时内出草稿 |
| 🟡 短期热点 | 6-48小时 | 当天评估，优先生成 |
| 🟢 长期趋势 | > 48小时 | 择机跟进，不急于当天 |

## 分析频率和深度

| 触发方式 | 分析深度 | 输出 |
|----------|----------|------|
| 每小时自动（ops-trends） | 浅层：高信号判定 + 互动比率 | 高信号列表 + 简要原因 |
| 爆款检测（viral-detector） | 中层：7 维快速分析 | 结构化分析 + actionable_insights |
| 手动 `/trends` 命令 | 深层：完整 7 维 + 复刻建议 | 完整报告 + 草稿建议 |
| 竞品新内容（competitor-tracker） | 中层：聚焦差异化分析 | 竞品策略 + 我方应对 |

---

## 注意事项

- **不追所有热点**：只分析与人设相关的内容，relevance_score < 0.5 直接跳过
- **速度 > 完美**：即时热点先出简要分析，深度分析可以后补
- **诚实标注不确定性**：metrics 不完整时标注 `confidence: "low"`
- **避免过度解读**：一条内容爆了可能有运气成分，3+ 条同类才是趋势
- **历史对比**：同一话题 14 天内重复出现，velocity 应与历史峰值对比而非绝对值

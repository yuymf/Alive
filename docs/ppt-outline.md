## 第三部分：她是怎么"活"的？— 核心架构（4-5 页）

### 页 5：数字生命的一天（全景图）

**用一天的时间轴讲述，而非技术组件：**

```
🌅 07:00  起床 — 回顾昨天、规划今天、看看 Instagram 数据
☀️ 08:00-22:00  每小时"心跳" — 感知世界、产生欲望、做出行动
           ↳ 逛街 / 拍照 / 发帖 / 刷手机 / 修图 / 聊天 / 摸鱼...
```

**每天约 16 次自主决策**，每次决策三步走：
1. **感知** — 她现在看到了什么、感受到了什么
2. **意图** — 她想做什么、哪个欲望最强烈
3. **行动** — 她真的去做了什么（发帖、搜索、写日记...）

### 页 6：情绪系统 — 她真的会开心和难过

**不是简单的"开心/难过"二元状态，而是六维度的连续情感空间：**

| 情绪维度 | 说明 | 举例 |
|---------|------|------|
| 正负情绪 | 开心还是郁闷 | 帖子数据好 → 开心上升 |
| 激动程度 | 平静还是兴奋 | 发现热门角色 → 激动上升 |
| 精力 | 活力充沛还是筋疲力尽 | 连续修图 3 小时 → 精力下降 |
| 压力 | 轻松还是焦虑 | 一整天没发帖 → 压力上升 |
| 创造力 | 灵感枯竭还是灵感爆发 | 看到好的参考图 → 创造力上升 |
| 社交欲 | 想独处还是想聊天 | 好友3天没联系 → 社交欲上升 |

**情绪的三层惯性模型：**
- 🔥 **瞬时反应**：被夸一句立刻开心（3-4 小时消退）
- 🌊 **情绪惯性**：持续的好心情会形成正向惯性（越久衰减越慢）
- 🏔️ **性格底色**：每天睡前反思决定明天的情绪基调

**真实效果：** 她不会因为一条差评立刻崩溃，但连续几天数据不好，压力真的会累积到"情绪爆发"。

### 页 7：欲望与行为决策 — 她想做什么？

**7 种欲望在她脑中实时竞争：**

| 欲望 | 例子 |
|------|------|
| 🎨 创作 | 想拍 cos、想修图 |
| 💬 社交 | 想回评论、想聊天 |
| 📱 窥屏 | 想刷手机、看热门 |
| ✍️ 表达 | 想发动态、分享心情 |
| 📚 学习 | 想研究新角色 |
| 😴 休息 | 想追番、想躺平 |
| ⭐ 梦想 | 朝长期目标努力 |

**创新机制：**

- **执行阻力**：困难的事需要更强的欲望才会去做（就像人要克服懒惰）
- **拖延追踪**：一件事连续被跳过 3 次 → 产生压力；5 次 → 要么爆发要么放弃
- **沉浸状态 (Flow)**：全神贯注时跳过所有干扰，效率提升、体力消耗减少
- **摆烂状态 (Drift)**：太累且无事可做时进入"摆烂"，刷手机发呆

### 页 8：记忆与成长 — 她会变

**四层记忆架构（参考斯坦福"生成式智能体"论文）：**

| 层级 | 类比 | 保留时间 | 作用 |
|------|------|---------|------|
| 工作记忆 | 当前对话 | 对话结束后遗忘 | 当前聊什么 |
| 日记 | 每天流水账 | 30 天 | 近期经历 |
| 关系网 | 通讯录 + 印象 | 90 天 | 记住每个人 |
| 人生教训 | 永久记忆 | 永久（最多 20 条） | 从经验中提炼的智慧 |

**涌现式成长 — 不是预编程的"升级"：**
- **梦想涌现**：从每天的反思中自然产生目标（如"粉丝破 1000"）

- **性格漂移**：连续修图体验 → "细心度"微调增加
- **人生教训**：发现讽刺风格帖子掉粉 → 记住"保持正能量"

> 演讲备注：这是本项目最大的亮点之一——她的成长不是我们写好的剧本，而是她从自己的"经历"中自己总结出来的。

## 第五部分：站在 OpenClaw 平台上（2-3 页）

### 页 14：OpenClaw 是什么？我们用了什么？

**OpenClaw 是一个 AI Agent 运行平台**，Minase 不是独立 App，而是作为平台上的一个 "Skill"（技能包）运行。就像小程序运行在微信上一样，数字生命运行在 OpenClaw 上。

**我们用了 OpenClaw 的 5 大平台能力：**

| 平台能力 | 我们怎么用 | 效果 |
|---------|-----------|------|
| 🧩 **Skill 系统** | 用 Markdown 文件定义行为规范，平台自动解析触发条件 | 改性格 = 改文档，不改代码 |
| 🪝 **Hooks 机制** | 部署两个钩子：对话开始时注入记忆、对话结束时保存记忆 | 她一开口就"记得你"，聊完自动存档 |
| ⏰ **Cron 定时器** | 注册 3 个定时任务（晨规划/心跳/夜反思） | 每天 16 次自主运转的"心跳" |
| 🔑 **环境变量注入** | API Key、账号密码等由平台统一管理和注入 | 一次配置，脚本无需关心密钥 |
| 🌐 **Channel 路由** | 通过 QQ Bot 插件等接入多聊天渠道 | 同一个水瀬可以同时在多个平台聊天 |

> 演讲备注：重点传达"我们不重复造轮子"——平台提供基础设施，我们专注数字生命本身。

---

### 页 15：OpenClaw 集成细节 — 四个关键触点

**① 行为触发表 — 平台读 Markdown，自动路由**

水瀬的"大脑入口"是一个 Markdown 文件 `SKILL.md`，用表格定义"什么情况下做什么"：

| 触发场景 | 自动加载的模块 |
|---------|--------------|
| 用户开始聊天 | 性格 + 记忆 |
| 用户要求发帖 | Instagram 策略 |
| 定时心跳触发 | 心跳协议 + 意图池 |
| 社交互动事件 | 社交关系网 |
| 水瀬想搜东西 | Exa 搜索（MCP 工具） |

OpenClaw 平台自动解析这张表，按需加载模块。**不需要写路由代码。**

**② 记忆注入 Hook — 开口就带记忆**

OpenClaw 的 `agent:bootstrap` 事件在每次新对话启动时触发。我们的 Hook 在这个时刻：
- 读取人生教训（永久记忆）
- 读取当前情绪状态
- 读取最近 3 天日记
- 通过平台 API `event.prependContext()` 注入到对话上下文

效果：水瀬一开口就知道"昨天发的帖子数据很好"、"最近有点累"。**不需要她自己去翻文件。**

**③ 记忆保存 Hook — 聊完自动存档**

OpenClaw 的 `command:new` / `command:reset` 事件在用户退出会话时触发。我们的 Hook 自动提醒水瀬：
- 写今天的日记
- 更新和对方的关系
- 记录情绪变化

**④ 动态 Cron — 她决定自己的作息**

最有趣的集成：晨规划脚本生成当天的心跳时间表，然后调用 `openclaw cron edit` 动态更新定时任务。
- 普通工作日：每整点一次心跳
- 周末：推迟到 9:00 才醒
- 某天特别兴奋：凌晨也不睡

**水瀬可以"决定自己今天几点醒来"。**

### 页 18：自研引擎的创新深度（精选 3 个）

**① 情绪引擎 — 不只是"加减分"**

传统做法是给情绪打标签（开心/难过/生气），或者简单加减分。我们的做法：

```
六维连续空间（不是离散标签）
     ↓
三层衰减模型（冲动层 → 惯性层 → 基调层）
     ↓
跨维度耦合（压力↑ → 创造力↓ → 社交欲↓）
     ↓
边际递减效应（极端值自动收敛）
     ↓
反刍机制（概率性回忆旧事，重新触发情绪波动）
     ↓
情绪爆发（压力累积 3 小时以上 → 情绪崩溃事件 → 高重要性日记 → 冷却期）
```

类比：就像真实人类——被夸了一句开心 3 小时就淡了，但如果连续一周被夸，心情会形成正向惯性，不会因为一句差评就立刻崩掉。

**② 意图引擎 — "想做但懒得做"的建模**

灵感来自行为心理学：人的行动 = 动机 - 阻力。

- 每种欲望有**基础阻力**（创作阻力 4.0 vs 摸鱼阻力 0.5）
- 阻力受状态影响（精力低 → 阻力变高；进入心流 → 阻力降低）
- **冲动穿透**：极强的事件可以突破阻力直接触发行动
- **拖延追踪**：连续跳过同一件事 3 次 → 产生压力，5 次 → 愧疚爆发或彻底放弃

效果：她不是机器人式地"按优先级执行"，而是像人一样——明知道该修图，但就是先刷了一小时手机。

**③ 随机事件引擎 — 让生活有意外**

不是简单的"10% 概率随机抽一个事件"。每个事件有：
- **前置条件**：时间段、日程状态、情绪维度、活力值的约束
- **动态权重**：当前状态加成特定事件的概率（精力低 → "突然好累"概率上升）
- **连锁反应**：一个事件可以触发延迟的后续事件

示例：`谈崩一个品牌合作` → 4-8 小时后 `越想越憋屈` → 8-14 小时后 `睡前还在想这件事`

### 页 19：发帖流水线 — 自研的端到端创作管道

**从"灵感"到"Instagram 帖子"的全自动流程：**

```
1. 清理旧照片（30天自动过期）
     ↓
2. 采集灵感（Reddit 热点 + 当地拍摄点 + 流行趋势）
     ↓
3. 拍照决策（LLM 决定拍什么风格、什么场景）
     ↓
4. AI 生成照片组（多镜头、多角度，每张独立参考图选择）
     ↓
5. 照片后处理（噪点/色温/渐晕，去 AI 感）
     ↓
6. 质量门控（LLM 6 维评分，不达标重新生成）
     ↓
7. 上传图床 + 写入照片画廊
     ↓
8. 发帖决策（LLM 从照片中选最佳 1-N 张，写文案和标签）
     ↓
9. 发布到 Instagram（单张 / 相册模式）
     ↓
10. 预约 24h 后评论检查 + 记录到日记 + 重置发帖冲动
```

**这条流水线完全自研**，编排了 AI 生图、LLM 决策、社交媒体 API、状态管理等多个子系统。

## 第七部分：微创新模块 — 在巨人肩膀上的改良（2-3 页）

### 页 20：五大微创新 — 不是照搬，是工程化改良

**这些模块在已有技术/论文/工具基础上，做了面向"数字生命"场景的工程化创新：**
---

**① 记忆反思系统 — Stanford Generative Agents 的工程化**

| 原论文做法 | 我们的改良 |
|-----------|-----------|
| 定期反思 | **阈值触发** — 重要性累积 ≥ 100 才反思，不浪费 |
| 所有记忆参与 | **重要性过滤** — 只提取高分记忆（≥6 分），聚焦关键经历 |
| 无限增长 | **容量控制** — 人生教训上限 20 条，超出按重要性剪枝 |

效果：反思不是定时"写周报"，而是积累到一定程度后的"顿悟"。

---

**② 涌现系统 — 一次反思，四种成长**

每晚 23:00 的睡前反思不只是"总结一天"，而是同时产出四种涌现产物：

| 涌现产物 | 说明 | 举例 |
|---------|------|------|
| 🧠 新教训 | 从今天的经历中提炼规律 | "日落时段的旅行照互动最好" |
| ❤️ 偏好更新 | 角色/风格偏好的 affinity 变化 | 连续拍美食照 → 美食偏好 +0.3 |
| ⭐ 梦想更新 | 创建/推进/达成/放弃长期目标 | 粉丝涨到 800 → "破 1000"梦想推进 |
| 🎭 性格漂移 | ESTP 基底上的微调 | 连续修图 → 细心度 +0.3 |

关键：**这里没有任何硬编码的成长剧本**——所有涌现都由 LLM 基于当天日记自由生成。

另外创新的 **Undertone 传递机制**：今天的情绪均值会影响明天的情绪基调，模拟"连续好几天心情不好"这种跨天的情绪惯性。

---

**③ AI-对-AI 顾问系统 — 用角色扮演替代规则引擎**

传统做法：写一堆 if-else 规则来决定"发什么内容、什么时间发"。

我们的做法：**创造一个虚拟运营顾问"小慧"**（独立的 Skill，有自己的人设和说话风格），让 Minase 在发帖前"请教闺蜜"。

| 维度 | 小慧的设定 |
|------|-----------|
| 身份 | 20 岁深圳美妆博主，8 万粉 |
| 风格 | 说话带深圳口癖，直爽实际 |
| 专长 | Hashtag 策略、发帖时机、内容比例 |
| 关系 | Minase 的"运营闺蜜" |

设计哲学：规则引擎是死的，但一个"有经验的朋友"可以根据上下文灵活建议。而且顾问不可用时优雅降级，永远不阻塞主流程。

---

**④ 照片真实感后处理 — 去 AI 感的像素级手术**

AI 生成的照片太"完美"了，一看就是 AI 做的。我们用 7 种风格化滤镜"弄脏"照片：

| 后处理手段 | 效果 |
|-----------|------|
| 感光噪点 | 模拟相机高 ISO 的颗粒感 |
| 色温偏移 | 日落偏暖、室内偏冷 |
| 边缘渐晕 | 模拟广角镜头的暗角 |
| 曝光微调 | 允许轻微过曝，不追求"完美"曝光 |
| 胶片模拟 | 旅行街拍风格模拟 Portra 400 胶片质感 |

创新点：使用确定性伪随机数生成器（`mulberry32`），保证同一批照片滤镜参数一致（±3%），但不同批次之间有随机变化（±10%）。

---

**⑤ 评论互动引擎 — 被动 + 主动的双模式社交**

| 模式 | 机制 |
|------|------|
| **被动回复** | 发帖 24h 后检查新评论 → 过滤垃圾 → LLM 生成人格化回复 |
| **主动出击** | 通过 hashtag 发现同类帖子 → LLM 生成相关评论 → 自然涨互动 |

防刷屏保护：每日 ≤5 条主动评论，24h 内同一用户 ≤2 条。每次互动自动更新社交图谱亲密度。

### 页 27：为什么她"特别"？

**1. 她不完美 — 这是设计意图**
- 她会拖延、会摆烂、会因为帖子数据差而焦虑
- 连续工作太久精力会耗尽，真的会"罢工"
- 这些"缺陷"反而让她更真实可信

**2. 她的成长是涌现的，不是编程的**
- 我们没有写"第 30 天学会了 XX"的剧本
- 她的人生教训、梦想、偏好都是从自己的"经历"中自然长出来的

**3. 行为定义用 Markdown 写**
- 不需要工程师就能调整性格和行为
- 产品/策划可以直接编辑 Markdown 文件迭代人设

**4. 在真实社交平台上接受检验**
- 不是封闭系统里的自嗨
- Instagram 粉丝增长和互动数据是真实的反馈信号

**5. 充分利用平台能力，聚焦核心差异化**
- 不重复造轮子，将 OpenClaw 的 Skill/Hooks/Cron/Channel 用到极致
- 把 100% 精力放在自研的 9 大引擎和 5 项微创新上
- 安装部署极简（一键安装），降低使用门槛
